#  Branching concepts

git branch    # Lists all branches in your project
git branch my-first-branch  (Branch name cannot have spaces) # Creates a new branch named my-first-branch
git branch my-second-branch (creates another branch)

git switch my-first-branch #switched to first branch

--Extra Notes  (create a new file or edit an existing one, and save your changes locally using your Phase 1 commands:
git add .git commit -m "Add new login automation scriptb"Note: This save only exists on your my-first-branch.
 If you switch back to main, this new file will literally vanish from your computer folder until you switch back!)

Merge changes back to main Once your new test script works perfectly and you want to combine it with your main project:
Switch back to your main trunk: git switch main Pull the changes into main: 

git merge my-first-branch

Delete the branch (Optional)Once merged, you no longer need the side branch:

git branch -d my-first-branch

-----------------------------------------------------------

To reinstate the deleted branch, use the command

git reflog  #USed to see complete history of every single action taken in the repo

find the commit hash

git checkout -b my-first-branch aec3350

----------------------------------------------------

To push the branch into GitHub, follow the command

git push --set-upstream origin my-first-branch

you only have to type that long command the very first time you push a new branch. for any future updates while working inside my-first-branch, we can simply do 

git push
